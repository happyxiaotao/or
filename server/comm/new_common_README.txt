redis和mysql都具有默认配置，在new的时候也接收配置

rediscli.lua脚本说明：
1，经过测试发现 一次连接+多次查询+一次断开 与 N次连接+多次查询+多次断开 耗时后者与前者的差距差不多，并且后者这种模式容易实现与维护
2，关于主从配置，在config.lua中进行配置，在new时，首先选择主库配置，在执行exec函数时，会修改默认配置为可以使用的配置（可能是主配置，可能是从配置）
3，关于redis中的锁的问题
	由于是每次查询都会进行重新连接，所以锁存在隐藏危险
	（比如在主库上加锁，下一次连接时，连接的是从库，直接使用）
	（比如在主库上解锁，却连接的是从库，解锁）
	在redis的配置中，保存上一次加锁的信息与下次解锁的信息做对比
